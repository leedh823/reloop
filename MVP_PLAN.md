# Reloop MVP 1단계 기획 정의서

## 📋 문서 목적
이 문서는 Reloop 앱 MVP 1단계의 제품 범위와 기능 정의를 명확히 하기 위한 기준 문서입니다.
라우팅 설계, UI/UX 디자인, 개발 작업의 기준으로 사용됩니다.

---

## 🎯 제품 정의

### 핵심 가치 제안
**Reloop은 실패를 기록하고, AI로 구조화해, 다시 시도하게 돕는 모바일 중심 앱이다.**

### 제품 성격
- **개인 저장소**: 공개 SNS가 아닌 개인 실패 기록 및 관리 도구
- **게스트 우선**: 회원가입 없이 바로 사용 가능
- **AI 기반 구조화**: 실패 경험을 AI가 분석하여 인사이트 제공

---

## ✅ MVP 1단계 포함 기능

### 1. 온보딩 → 게스트 진입
- 앱 첫 진입 시 간단한 소개
- 게스트 모드로 즉시 시작 가능

### 2. 실패 작성 (단계형)
- 단계별 폼으로 실패 경험 기록
- 필수 정보: 제목, 요약, 상세 내용, 카테고리, 감정 태그
- 선택 정보: PDF 파일 첨부

### 3. 실패 목록/상세
- 작성한 실패 목록 조회
- 실패 상세 내용 확인
- 카테고리/감정 태그 필터링

### 4. AI 분석
- 실패 기록을 AI에게 전달
- AI가 감정과 내용을 구조화하여 분석
- 분석 결과를 텍스트로 제공

### 5. AI 결과 저장
- AI 분석 결과를 실패 기록에 연결하여 저장
- 이후 조회 시 분석 결과 함께 확인 가능

## 📱 화면(Screen) 목록

### MVP 유저 플로우 기반 화면 구성

**플로우 A**: 온보딩 → 실패 작성 → AI 분석  
**플로우 B**: 실패 목록 → 실패 상세 → AI 분석

---

### 1. 온보딩 화면 (Onboarding Screen)

**목적**: 앱의 가치 제안을 간단히 소개하고, 게스트 모드로 진입 유도

**주요 액션 (CTA)**:
- "시작하기" 버튼 → 게스트 모드로 진입

**입력 데이터**: 없음

**출력 데이터**: 
- 게스트 모드 진입 여부 (로컬 스토리지/세션)

**화면 구성 요소**:
- 앱 로고/브랜딩
- 핵심 가치 제안 텍스트 (1-2문장)
- "시작하기" 버튼

---

### 2. 홈 화면 (Home Screen)

**목적**: 실패 목록을 카드 형태로 표시하고, 새 실패 작성 진입점 제공

**주요 액션 (CTA)**:
- "실패 작성하기" 버튼 (FAB 또는 하단 탭)
- 실패 카드 클릭 → 실패 상세 화면 이동

**입력 데이터**: 없음 (로컬에서 실패 목록 조회)

**출력 데이터**:
- 실패 목록 (제목, 요약, 카테고리, 감정 태그, 작성일)
- 필터링된 실패 목록 (카테고리/감정 태그 기준)

**화면 구성 요소**:
- 검색바 (제목/요약 검색)
- 필터 (카테고리, 감정 태그)
- 실패 카드 리스트
- "실패 작성하기" FAB

---

### 3. 실패 작성 화면 (Compose Failure Screen)

**목적**: 단계별 폼을 통해 실패 경험을 구조화하여 기록

**주요 액션 (CTA)**:
- "저장하기" 버튼 → 실패 저장 후 상세 화면 이동
- "취소" 버튼 → 이전 화면으로 돌아가기

**입력 데이터**:
- 제목 (필수, 텍스트)
- 요약 (필수, 텍스트)
- 상세 내용 (필수, 텍스트)
- 카테고리 (필수, 선택)
- 감정 태그 (필수, 선택)
- PDF 파일 (선택, 파일 업로드)

**출력 데이터**:
- 저장된 실패 기록 (ID, 작성일시 포함)
- 저장 성공/실패 상태

**화면 구성 요소**:
- 단계별 입력 폼
- 필수/선택 표시
- 파일 업로드 영역
- 저장/취소 버튼

---

### 4. 실패 상세 화면 (Failure Detail Screen)

**목적**: 저장된 실패 기록의 전체 내용을 확인하고, AI 분석 진입점 제공

**주요 액션 (CTA)**:
- "AI 분석하기" 버튼 → AI 분석 화면 이동
- "목록으로" 버튼 → 홈 화면 이동

**입력 데이터**: 실패 ID (URL 파라미터)

**출력 데이터**:
- 실패 상세 정보 (제목, 요약, 상세 내용, 카테고리, 감정 태그, 작성일)
- AI 분석 결과 존재 여부 (있으면 표시)

**화면 구성 요소**:
- 실패 제목
- 카테고리/감정 태그 배지
- 요약 섹션
- 상세 내용 섹션
- PDF 파일 링크 (있는 경우)
- AI 분석 결과 미리보기 (있는 경우)
- "AI 분석하기" 버튼

---

### 5. AI 분석 화면 (AI Analysis Screen)

**목적**: 실패 기록을 AI에게 전달하고, 분석 결과를 받아서 표시

**주요 액션 (CTA)**:
- "분석 요청하기" 버튼 → AI 분석 API 호출
- "결과 저장하기" 버튼 → 분석 결과를 실패 기록에 연결하여 저장

**입력 데이터**:
- 실패 기록 ID (기존 실패 분석 시)
- 또는 실패 작성 폼 데이터 (작성 직후 분석 시)
- 추가 컨텍스트 (선택, 텍스트)

**출력 데이터**:
- AI 분석 결과:
  - 요약 (텍스트)
  - 감정 분석 (텍스트)
  - 제안 질문 (텍스트 리스트)
- 분석 상태 (로딩/완료/실패)

**화면 구성 요소**:
- 분석 대상 실패 정보 표시
- 추가 컨텍스트 입력 (선택)
- "분석 요청하기" 버튼
- 로딩 인디케이터
- 분석 결과 표시 영역:
  - 요약
  - 감정 분석
  - 제안 질문
- "결과 저장하기" 버튼

---

### 6. AI 분석 결과 화면 (AI Analysis Result Screen)

**목적**: 저장된 AI 분석 결과를 상세히 확인

**주요 액션 (CTA)**:
- "다시 분석하기" 버튼 (선택) → AI 분석 화면으로 이동
- "목록으로" 버튼 → 홈 화면 이동

**입력 데이터**: 실패 ID (URL 파라미터)

**출력 데이터**:
- 저장된 AI 분석 결과 (요약, 감정 분석, 제안 질문)
- 분석 일시

**화면 구성 요소**:
- 실패 제목 (참고용)
- AI 분석 결과 섹션:
  - 요약
  - 감정 분석
  - 제안 질문
- "다시 분석하기" 버튼 (선택)

## 🔄 유저 플로우 상세

### 플로우 A: 온보딩 → 실패 작성 → AI 분석

1. **온보딩 화면**
   - 사용자: "시작하기" 클릭
   - 시스템: 게스트 모드 활성화, 홈 화면 이동

2. **홈 화면**
   - 사용자: "실패 작성하기" 클릭
   - 시스템: 실패 작성 화면 이동

3. **실패 작성 화면**
   - 사용자: 폼 입력 후 "저장하기" 클릭
   - 시스템: 실패 저장, 실패 상세 화면 이동

4. **실패 상세 화면**
   - 사용자: "AI 분석하기" 클릭
   - 시스템: AI 분석 화면 이동 (실패 ID 전달)

5. **AI 분석 화면**
   - 사용자: "분석 요청하기" 클릭
   - 시스템: AI 분석 API 호출, 결과 표시
   - 사용자: "결과 저장하기" 클릭
   - 시스템: 분석 결과 저장, 실패 상세 화면 이동 (분석 결과 표시)

---

### 플로우 B: 실패 목록 → 실패 상세 → AI 분석

1. **홈 화면**
   - 사용자: 실패 카드 클릭
   - 시스템: 실패 상세 화면 이동

2. **실패 상세 화면**
   - 사용자: "AI 분석하기" 클릭
   - 시스템: AI 분석 화면 이동 (실패 ID 전달)

3. **AI 분석 화면**
   - 사용자: "분석 요청하기" 클릭
   - 시스템: AI 분석 API 호출, 결과 표시
   - 사용자: "결과 저장하기" 클릭
   - 시스템: 분석 결과 저장, 실패 상세 화면 이동 (분석 결과 표시)

## 📊 데이터 모델 (간단 정의)

### 실패 기록 (Failure)
- `id`: 고유 식별자
- `title`: 제목
- `summary`: 요약
- `content`: 상세 내용
- `category`: 카테고리
- `emotionTag`: 감정 태그
- `pdfUrl`: PDF 파일 URL (선택)
- `createdAt`: 작성일시
- `aiAnalysisId`: AI 분석 결과 연결 ID (선택)

### AI 분석 결과 (AIAnalysis)
- `id`: 고유 식별자
- `failureId`: 연결된 실패 기록 ID
- `summary`: AI 요약
- `emotionAnalysis`: 감정 분석
- `suggestedQuestions`: 제안 질문 리스트
- `createdAt`: 분석일시

## ✅ MVP 완료 기준

다음 조건을 모두 만족하면 MVP 1단계 완료:

1. ✅ 게스트 모드로 앱 진입 가능
2. ✅ 실패 작성 → 저장 → 목록 조회 가능
3. ✅ 실패 상세 확인 가능
4. ✅ 실패 기록을 AI에게 전달하여 분석 가능
5. ✅ AI 분석 결과를 실패 기록에 연결하여 저장 가능
6. ✅ 저장된 분석 결과를 실패 상세에서 확인 가능
